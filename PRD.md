## 1. 概述

### 1.1 产品定位

**一句话**：让 AI 替你记住关键的人、看见重要的事、活出想成为的样子。

**本质**：从「社交副驾驶」升级为「**生活副驾驶**」——
- 旧：处理消息和关系的工具
- 新：以「身份 → 主线 → 关系 + 时间」三轴为骨架的个人生活操作系统

完整战略说明见 [LIFE_COPILOT.md](LIFE_COPILOT.md)。

### 1.2 三轴框架

```
身份（北极星）→ 主线（3-5 条 30-90 天方向）→ 关系 + 时间 → 今日 3 件事 → 复盘 → 喂回身份
```

每个产品功能必须能映射回这条主链。任何功能如果不能解释"它如何把用户向北极星推进 1 步"，就不该做。

### 1.3 目标用户

| Persona | 痛点 | 核心需求 |
|---------|------|---------|
| **A. 社交超载者**（创业者/PM） | 好友 500+，记不住谁是谁，每天被消息推着走 | 关系沉淀 + 主线视角下的优先级 |
| **B. 关系经营者**（销售/HR） | 必须主动维护人脉 | 主动提醒 + 起草专业回复 + 同路人 |
| **C. 社交疲惫者**（普通白领） | 不想敷衍但必须回应 | 一键应和 + 复盘闭环重塑节奏 |
| **D. 自我成长者**（学习者/转型者）| 知道想变好，但不知道今天该做什么 | 北极星 + 主线 + 同路人 + 每日复盘 |

v1.0 主攻 Persona A、C、D（消费场景）。v1.4+ 扩展 B（B 端）。

### 1.4 核心价值主张

| 旧（Social Copilot） | 新（Life Copilot） |
|---------------------|---------------------|
| 1. 降低社交关系管理成本 | 1. 给每天一个清晰的方向感 |
| 2. 降低社交仪式成本 | 2. 让每段关系成为推动主线的资源（而不是消耗） |
|  | 3. 让今天的每个动作和身份对齐 |
|  | 4. 用复盘把今天的经验复利成明天的能力 |

### 1.5 差异化

| 竞品 | 局限 | Pulse 的优势 |
|------|------|------------|
| ChatGPT / Claude | 无人格连续性、无关系记忆、不进入聊天流、无身份连续性 | 有名字（小 P）+ 长期偏好图谱 + 嵌入聊天 + 身份与主线的连续记忆 |
| 微信 | 无 AI、无关系画像、无主线视角 | 在微信之上加一层 Life Copilot |
| Notion / Sunsama / Things | 任务管理工具，不懂关系 | 关系 × 主线 × 时间三轴一体 |
| Monica AI / Clay / Dex | CRM 类，不进入对话流，不懂个人主线 | 副驾驶 + 主线视角 + 同路人 |
| 「地球 online」类成长产品 | 仅主线，缺关系层 | 主线与关系深度融合，关系成为主线的杠杆 |

**核心壁垒**：长期沉淀的「身份 + 主线 + 偏好 + 关系」四层图谱——用户用越久，离不开越深。

### 1.6 1 Day 哲学的产品翻译

参考「1 Day Change Your Entire Life」的 5 条原则，本产品的核心规则：

| 原则 | 产品规则 |
|------|---------|
| Today Is Enough | 主屏只展示今天，长期目标不做主信息架构 |
| Identity > Goals | 身份层是北极星，所有功能都要服务身份 |
| Daily 3 | 每天三件事是上限——不允许 4+ |
| Connection Compounds | 关系健康度从警告型升级为复利型 |
| Reflection Closes Loop | 复盘是小 P 长期记忆**唯一**的写入入口 |

## 2. 用户故事与场景

### 2.1 Persona A 的一天（创业者，Life Copilot 视角）

| 时刻 | 场景 | Pulse 介入 |
|------|------|-----------|
| 7:30 早晨 | 打开 App | 今日驾驶舱：北极星常驻 + 今日 3 件事（写完支付 OKR / 19:30 陪女儿 / 跑 3km）+ 待回 3 条 + 今日值得连接（妈妈 28 天） |
| 9:30 通勤 | 投资人 Ava 催 Q3 合同 | 选 hint「承诺时间」→ 草稿区上方："对主线 A 是助力 · 主线关键节点" → 5 秒发出 |
| 11:00 工作 | 群聊 124 条 | 小 P 摘要：v3 已成基线，你需周五前定 review 时间（标 "🟠 主线 A"） |
| 14:00 午饭后 | Mira 邀约 8pm | 跟小 P 商量 → 推荐理由 3 条，第 3 条来自身份层："Mira 是你测试『值得信赖的朋友』身份的最佳对象" → 采用「热情确认」，且把阅读改到明早 |
| 18:00 下班 | Jules 宣布开工作室 | 跟小 P 商量 → 推荐「支持+庆祝」+ "对你的身份『影响 100 万产品人的设计师』有共振" → 一键采用 |
| 22:00 睡前 | 推送复盘 | 三件事 2/3 完成 → 跑步延期 → 选「太忙」→ 小 P："本周第二次延期主线 B，要不要把强度先调到 2km？" + 明日草拟 |

### 2.2 Persona D 的关键场景（自我成长者，Life Copilot 全功能展示）

| 场景 | 体验 |
|------|------|
| **首启** | 小 P 问"你想成为什么样的人？" → 用户写"成为能影响 100 万产品人的设计师" → 小 P："好，这是你的北极星。我会帮你把每一天对齐到这里" |
| **首周** | 小 P 不主动推主线，先观察对话 / 行为 7 天 → 第 8 天给出"我看到你最近频繁聊『OKR』『产品』，要不要把『转型 PM』作为主线 A？" |
| **第一次复盘** | 22:00 推送 → 三件事自评 → 小 P 反馈"今天主线 C 完成质量很高，感觉到你和女儿的连接更深了" |
| **同路人解锁** | 主线 A 稳定 2 周后 → 小 P："你的主线和 K 高度对齐，两人都在 PM 转型第 30-60 天，要不要 15 分钟轻交流？" → 用户接受 → 弹通话 |
| **身份微调** | 1 个月后小 P："观察到你最常说的不是『100 万产品人』，是『让初级 PM 不再走弯路』。要不要把北极星改写为更精准的版本？" |

### 2.3 Persona C 的关键场景（普通白领）

| 场景 | 体验 |
|------|------|
| **同事生日** | 小 P 主动准备 30 个朋友的生日草稿，每条都不重样（v1.3） |
| **客户反馈** | 草稿区点 ✅ 承诺时间 → 简练专业 → 发出 |
| **家人健康** | 爸爸提复查 → 小 P 看出"他真需要陪同" → 标"对主线 C『家人陪伴』强助力" → 采纳「主动陪同」 |
| **立场分歧** | Noa 发争议话题 → 小 P 提供「温和分歧」+「转移话题」 + 警告"不能敷衍" |
| **被动复盘养成** | 第 1 周用户跳过复盘 4/7 天 → 小 P："要不要把复盘时间从 22:00 改到 21:00？或者只问一个问题？" |

### 3.1 ★ 今日驾驶舱（Today Cockpit）— NEW HOME

**模块编号**：M8

**描述**：取代旧的「智能收件箱」作为主屏。一打开 App 就看见**身份 + 主线 + 关系 + 时间**的完整今日切片。

#### 功能列表

| ID | 功能 | 状态 | 优先级 |
|----|------|------|--------|
| 1.1 | 顶部北极星常驻（一行身份陈述） | ○ | **P0** |
| 1.2 | 时间感知开场（早 / 中 / 晚 / 夜，5 时段语调不同） | ◐ | **P0** |
| 1.3 | 今日 3 件事（来自主线池，用户可改） | ○ | **P0** |
| 1.4 | 三件事完成态切换（待办 / 完成 / 延期 + 原因） | ○ | **P0** |
| 1.5 | 今日值得连接（1 个，复利型推荐） | ○ | **P0** |
| 1.6 | 待回消息预览（≤ 5 条，链接到收件箱）| ○ | **P0** |
| 1.7 | 「跟小 P 商量今天」总入口按钮 | ○ | **P0** |
| 1.8 | 时间块视图（早 / 上午 / 下午 / 晚 / 夜，可拖拽三件事） | ○ | **P1** |
| 1.9 | 「我今天没方向」紧急按钮（小 P 直接给 1-3 件事） | ○ | **P1** |
| 1.10 | 今日小结浮层（22:00 自动推复盘入口） | ○ | **P0** |

#### 输入

- 身份（北极星）
- 主线池（今日所有需要推进的主线动作）
- 待回消息（来自 M1 收件箱）
- 关系健康度（来自 M11，复利模式）
- 用户精力曲线（来自 M14 长期记忆）

#### 输出

```typescript
{
  northStar: "成为最值得信赖的爸爸 + 影响 100 万产品人的设计师",
  greeting: { period: "morning", line: "早安 Ethan" },
  daily3: [
    { mainline: "A", title: "写完支付模块 OKR", energy: 90, slot: "上午" },
    { mainline: "C", title: "19:30 陪女儿讲故事", energy: 30, slot: "晚间" },
    { mainline: "B", title: "跑 3km", energy: 60, slot: "下午" }
  ],
  todayConnect: { contactId: "mom", reason: "28 天未联系", action: "1 分钟语音" },
  inboxPreview: { now: 3, later: 2, ignore: 4 },
  reflectionDue: { time: "22:00", enabled: true }
}
```

#### 验收标准

- **AC-1.1**：Given 用户首次打开（已设北极星）/ When 加载主屏 / Then 北极星 + 今日 3 件事 + 待回消息在 1 秒内同框呈现
- **AC-1.2**：Given 用户改写今日三件事 / When 删除一件 / Then 提示"小 P 建议保留这件——它对主线 A 是关键节点"，但**用户可强制删**（不能阻止）
- **AC-1.3**：Given 完成一件事 / When 点击 ✓ / Then 触发学习信号 + 三件事计数 +1
- **AC-1.4**：Given 22:00 / When 用户在 App 中 / Then 自动滑出"今日复盘"入口（用户可一键推迟）
- **AC-1.5**：Given 用户点「我今天没方向」 / When 触发 / Then 小 P 在 2 秒内给出 1-3 件可执行事

#### 依赖

M6（北极星）+ M7（主线池）+ M11（关系复利）+ M14（长期记忆）

### 3.2 ★ 北极星（North Star · Identity）

**模块编号**：M6

**描述**：用户的身份陈述——"我想成为什么样的人"。Pulse 一切建议都以此为最终判据。

#### 功能列表

| ID | 功能 | 状态 | 优先级 |
|----|------|------|--------|
| 2.1 | 首启时引导用户写北极星（含 3 个范例） | ○ | **P0** |
| 2.2 | 「先用 2 周再回答」延迟选项 | ○ | **P0** |
| 2.3 | 北极星陈述自由文本编辑（≤ 200 字） | ○ | **P0** |
| 2.4 | 历史版本（修改时旧版自动归档） | ○ | **P1** |
| 2.5 | 小 P 建议改写（基于 30 天观察） | ○ | **P1** |
| 2.6 | 北极星与主线的关联可视化 | ○ | **P1** |
| 2.7 | 多版本（工作我 / 家庭我 / 学习我）| ○ | **P3** |

#### 输入

- 用户首启输入 / 后续编辑
- 30 天的对话 + 复盘 + 主线动作（小 P 用来反推用户真实身份取向）

#### 输出

```typescript
{
  current: { text: "...", createdAt: "2026-06-02", confidence: 0.7 },
  history: [{ text, createdAt, archivedReason }, ...],
  pulseObservation?: {
    suggestedRevision: "...",
    evidence: ["对话 #1234 提到...", "复盘 #45 写了..."]
  }
}
```

#### 验收标准

- **AC-2.1**：Given 首启 / When 提示输入北极星 / Then 提供"先用 2 周"按钮，可跳过
- **AC-2.2**：Given 已设北极星 30 天 / When 小 P 完成观察 / Then 主面板出现"基于观察的北极星修订建议"卡片，**仅建议不强制**
- **AC-2.3**：Given 用户改北极星 / When 保存 / Then 旧版自动归档，**不删除**

#### 依赖

M14（长期记忆）

### 3.3 ★ 我的主线（Mainlines）

**模块编号**：M7

**描述**：用户当前活跃的 3-5 条 30-90 天主线。每条都连回北极星，并被分解到今日动作。

#### 功能列表

| ID | 功能 | 状态 | 优先级 |
|----|------|------|--------|
| 3.1 | 主线池视图（卡片列表） | ○ | **P0** |
| 3.2 | 单条主线详情（动机 / 完成标准 / 进度 / 时间窗 / 历史动作） | ○ | **P0** |
| 3.3 | 新建主线（含「为什么」必填字段，连回北极星）| ○ | **P0** |
| 3.4 | 编辑 / 暂停 / 完成 / 放弃主线（放弃需选原因）| ○ | **P0** |
| 3.5 | 主线下今日动作（≤ 3 个/天/主线） | ○ | **P0** |
| 3.6 | 进度条 + 连续达成天数 🔥 | ○ | **P0** |
| 3.7 | 小 P 主动建议主线（基于观察 7-14 天）| ○ | **P1** |
| 3.8 | 同路人入口（链接 M10）| ○ | **P2** |
| 3.9 | 主线模板库（PM 转型 / 副业 / 健身 / 育儿等通用模板）| ○ | **P3** |
| 3.10 | 主线之间冲突检测（"主线 A 时间需求 vs 主线 B 时间需求"）| ○ | **P2** |

#### 输入

- 用户主动定义的主线 / 小 P 建议
- 北极星陈述
- 关系画像（关联到 M1 模块 1）
- 复盘记录（来自 M9）

#### 输出

```typescript
{
  mainlines: [
    {
      id: "A",
      title: "转型 PM",
      motivation: "服务身份『设计师 → 产品人』",
      completionCriteria: "30 天通过 OKR 模板做完 5 个真实需求",
      window: { start: "2026-05-01", end: "2026-05-31" },
      progress: 0.4,
      todayActions: [{...}],
      streak: 0,
      relatedContacts: ["mira", "k"]
    },
    ...
  ]
}
```

#### 验收标准

- **AC-3.1**：Given 用户新建主线 / When 提交 / Then 必填「为什么」字段，且系统提示其与北极星的关联
- **AC-3.2**：Given 主线 ≥ 6 条 / When 用户尝试新建第 6 条 / Then 提示"超过推荐上限 5 条，主线太多反而失焦，建议先归档一条"——**仅提示不阻止**
- **AC-3.3**：Given 主线放弃 / When 用户操作 / Then 必选原因（路径偏了 / 已不重要 / 太难 / 其他），数据流入 M14 学习
- **AC-3.4**：Given 主线 30 天进度 < 10% / When 系统检出 / Then 小 P 主动询问"这条主线最近没动，还要继续吗？"

#### 依赖

M6（北极星）+ M14（长期记忆）+ M9（复盘）

### 3.4 ★ 每日复盘（Daily Reflection）

**模块编号**：M9

**描述**：每晚 5 分钟轻量复盘——是小 P 长期记忆**唯一**的写入入口。

#### 功能列表

| ID | 功能 | 状态 | 优先级 |
|----|------|------|--------|
| 4.1 | 22:00 默认推送（用户可改时间，可关）| ○ | **P0** |
| 4.2 | 三问表单（三件事 / 最好的互动 / 明日三件事）| ○ | **P0** |
| 4.3 | 未完成事项原因选择（4 个选项 + 其他）| ○ | **P0** |
| 4.4 | 小 P 即时反馈（≤ 1 段话）| ○ | **P0** |
| 4.5 | 复盘历史回看（近 7 天 / 近 30 天）| ○ | **P1** |
| 4.6 | 跳过 3 天后小 P 主动询问（不指责）| ○ | **P0** |
| 4.7 | 连续 30 天的庆祝 + 简化为 2 问的选项 | ○ | **P1** |
| 4.8 | 周维度模式（仅周日复盘上一周，可替代日复盘）| ○ | **P2** |
| 4.9 | 复盘内容**唯一**长期记忆写入入口 | ○ | **P0** |

#### 输入

- 当日完成 / 未完成的三件事
- 当日所有对话（小 P 已读取）
- 当日所有关系互动
- 用户主动文本输入

#### 输出

```typescript
{
  date: "2026-06-02",
  daily3Outcome: [
    { id, status: "done"|"deferred"|"abandoned", reason?: string },
    ...
  ],
  bestMoment: "Mira 跟我聊她的 PM 转型，让我感觉不孤单",
  tomorrowDaily3: [...],
  piResponse: "...",
```

| 主动推送可关 | 早简报 / 复盘 / 同路人 / 关系提醒 4 项必须各自可关 |
| 复盘绝不指责 | 跳过 / 拖延的复盘场景仅询问，不评价 |
| 不游戏化 | 没有 XP / 等级 / 排行榜 / 徽章成就（连续达成 🔥 是唯一例外，但不可分享） |
| 主线可暂停 / 放弃 | 用户随时可暂停 / 放弃任何主线，无后果 |
| 北极星可改 | 可任意修改，旧版自动归档不删除 |

### 5.2 用户体验验收

### 5.4 隐私验收

## 8. 风险与缓解

| 风险 | 概率 | 影响 | 缓解策略 |
