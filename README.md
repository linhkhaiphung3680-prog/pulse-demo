## 这是什么

Pulse 是一款 AI 生活副驾驶（Life Copilot）的高保真原型。从早期的「社交副驾驶」演进而来，现在它是「**个人生活操作系统**」——把身份、主线、关系、时间放在一个统一的视角下管理。

它解决三个真实痛点：

1. **没方向感**：用户在每条消息间被推着走，不知道今天为什么做这件事
2. **关系与目标脱节**：关系是消耗，还是助力？没系统能告诉你
3. **社交仪式成本**：应和、确认、客套不再需要手打字

AI 在产品里的化身叫**小 P**（Pi）——有名字、有时间感、有可被纠正的偏好、有可分层的风格、且**陪你跑「身份 → 主线 → 今天 → 复盘」的闭环**。这是 ChatGPT 类工具型产品做不到的，因为它们没有人格连续性和身份连续性。

---

## 三轴框架

```
身份（北极星）→ 主线（3-5 条 30-90 天）→ 关系 + 时间 → 今日 3 件事 → 复盘 → 喂回身份
```

每个功能都从这条主链衍生。完整战略说明见 [LIFE_COPILOT.md](LIFE_COPILOT.md)。

参考方法论：「1 Day Change Your Entire Life」5 原则——
- Today Is Enough（今天就够）
- Identity > Goals（身份重于目标）
- Daily 3（每天三件事）
- Connection Compounds（连接会复利）
- Reflection Closes the Loop（复盘闭合循环）

## 9 个体验切片

> ⊕ = 升级自原社交副驾驶版本　★ = Life Copilot 全新模块

### ⊕ ① 今日驾驶舱（NEW HOME）
取代旧的「智能收件箱」做主屏。一打开 App 就同框看到：
- 顶部北极星常驻（你的身份陈述）
- 今日三件事（来自主线池，时间感知开场）
- 今日值得连接（1 个，复利型推荐）
- 待回消息预览（链接到收件箱）
- 「跟小 P 商量今天」总入口

### ⊕ ② 智能社交收件箱
不再按时间排序，按**小 P 推断的优先级**分组：立刻回 / 稍后回 / 可忽略。
**升级**：每条消息加**主线徽章**——这条消息影响哪条主线 / 哪段关系。

### ⊕ ③ 聊天 + 副驾驶
打开任意联系人，下半屏出现**小 P 看完了**面板：
- 2 个**动态 hint**（基于关系 × 对方话语生成）
- 选中 hint 后弹出草稿，下方是**场景化 chips**
- 自定义指令（fallback）
- **升级**：草稿区上方一行**主线影响标**——「对你的主线 A 是助力 / 中性 / 拖累」

### ⊕ ④ 跟小 P 商量这条
草稿区点「跟小 P 商量」打开 sheet——小 P 说出：
- 她到底想说什么（intent）
- 她真正期待（expectations）
- 历史里值得看的（带具体日期的历史信号）
- ⚠️ 风险点
- ⭐ **小 P 的建议**：明确推荐选哪个 hint，附 3 条理由
- **升级**：3 条理由里**至少 1 条来自身份/主线层**

### ⊕ ⑤ 关系网络
SVG 同心圆图谱，按关系强度分 3 圈，按社群分 4 色。
**升级**：节点加**主线关联徽章** + 关系健康度从警告型升级为**复利型**（"再问候 1 次，连接强度从 32% → 58%"）。

### ★ ⑥ 我的主线（NEW）
底部 tabbar 第 4 位新增。维护 3-5 条 30-90 天主线（每条带：动机连回身份 / 完成标准 / 时间窗 / 进度 / 今日动作 / 同路人入口）。

### ★ ⑦ 同路人发现（NEW · 弱触发）
来自「地球 online」模型。**默认关闭**，主线稳定 ≥ 2 周后才解锁，仅推送主线对齐 + 进度阶段相近的人。从「15 分钟轻交流」开始，**不互加好友**。

### ★ ⑧ 每日复盘（NEW）
每晚 22:00 推送。5 分钟内完成的三问：
1. 今天三件事做完了吗？没做完是因为什么？
2. 今天哪一次互动让你感觉最好？
3. 明天的三件事是什么？

复盘是小 P 长期记忆**唯一**的写入入口——白天的小 P 是助理，夜晚的小 P 是教练。

### ⊕ ⑨ 小 P 主面板
原 4 tab 扩到 **6 tab**：北极星 / 主线 / 对话 / 风格 / 权限 / 已学到。
权限 tab 新增「**小 P 可主动推送的内容**」分级开关——早简报 / 复盘 / 同路人 / 关系提醒可单独开关。

## 视觉设计

| 元素 | 选择 | 理由 |
|------|------|------|
| 主背景 | `#0B0C10` 暗色 | 让 AI 紫色更突出 |
| AI 主色 | Pulse 紫 `#8B5CF6` | 与微信绿形成对比，让 AI 元素一眼辨认 |
| 自己的消息气泡 | 微信绿 `#95EC69` | 中文用户的肌肉记忆 |
| 对方消息气泡 | 白色 + 尖角 | 严格还原微信 UI |
| 小 P 头像 | 紫色渐变圆形 + 数学符号 π（斜体 serif） | 辨识度极强，单字母可缩放 |
| 设备外壳 | iPhone 边框 + 灵动岛 | 让 demo 像真实 App 而非网页 |

## 技术栈

```
单 HTML 文件
├── Tailwind CSS（CDN，含自定义颜色 token）
├── 内嵌 CSS（动画 / 微信气泡 / 关系图样式）
├── 内嵌 JavaScript（无外部依赖）
│   ├── 数据：CONTACTS / INBOX / AI_REPLIES / PI_CTX_PRESETS
│   ├── 路由：5 屏间切换（v1.0 起扩到 9 屏）
│   ├── Pi 模拟：Wizard of Oz（关键词匹配预设响应）
│   └── 关系图：纯 SVG，无 D3
└── 169 KB · 2900 行 · 0 后端
```

**为什么单文件**：让 demo 真正「即开即用」——双击就能本地跑，分享一个 URL 任何人就能体验，0 安装 0 配置 0 后端。

**为什么 Wizard of Oz**：v0.x 阶段我们要解决的是**产品定义问题**而非**技术问题**。先用预设脚本把"理想 AI 体验"做出来，再去想怎么让真实 LLM 达到这个体验——而不是反过来。

## 产品决策的"为什么"

每个 UX 选择背后都有可解释的产品判断，例如：

- 为什么从「四态固定按钮」改为「2 个动态 hint」
- 为什么 chips 是主路径，文字指令只是 fallback
- 为什么「忘掉」必须给原因
- 为什么有了"小 P"就不能再用"AI"作为主语
- 为什么风险点要用黄色而非红色
- 为什么数据"仅在本地"是不可关闭的开关
- **为什么从社交副驾驶升级为生活副驾驶**（v0.4-pre 决策 #17）
- **为什么小 P 的主动推送必须分级可关**（决策 #18）
- **为什么同路人默认关闭 + 双向同意**（决策 #19）
- **为什么身份用「北极星」而不是「目标」/「Vision」**（决策 #20）

详见 [PRODUCT_NOTES.md](PRODUCT_NOTES.md)。

## 这个 demo 不做的事

- **真实的微信对接 / 数据导入**：demo 假设我们已有微信全量上下文
- **真实的 LLM 调用**：用 Wizard of Oz 预设脚本模拟
- **用户登录 / 账户系统**
- **桌面端响应式**：移动端 frame 居中即可
- **同路人匹配的真实运行**：v0.7 才上线，且需要法律咨询 + 隐私白皮书

这些都在 [ROADMAP](ROADMAP.md) 里，但本 demo 阶段聚焦于「**让产品的形与魂先长出来**」。
